# Deployment Overview

Scrapix supports two deployment modes across many platforms. This guide helps you choose.

## Decision Matrix

| Platform | Mode | Kafka? | Min RAM | Approx. Cost | Best For |
|----------|------|--------|---------|------------|---------|
| [Docker Compose](/deployment/docker-compose) | Both | Optional | 2 GB | Free (self-hosted) | Local dev, small production |
| [Kubernetes](/deployment/kubernetes) | Distributed | Yes | 8 GB | Varies | Large-scale, auto-scaling |
| [Fly.io](/deployment/fly-io) | Distributed | Yes | 2 GB | ~$30/mo | Multi-region, edge |
| [Railway](/deployment/railway) | Single binary | No | 512 MB | ~$5/mo | Quick deploys, hobby |
| [Render](/deployment/render) | Single binary | No | 512 MB | ~$7/mo | Simple PaaS |
| [Google Cloud Run](/deployment/google-cloud-run) | Single binary | No | 512 MB | Pay-per-use | Scale-to-zero, bursty |
| [AWS](/deployment/aws) | Both | Optional | 1 GB | ~$30/mo+ | Enterprise, existing AWS |
| [Azure](/deployment/azure) | Single binary | No | 1 GB | ~$20/mo | Enterprise, existing Azure |
| [Hetzner VPS](/deployment/hetzner-vps) | Single binary | No | 2 GB | ~EUR 4.50/mo | Budget, European hosting |
| [Coolify](/deployment/coolify) | Single binary | No | 2 GB | Free (self-hosted) | Self-hosted PaaS |

## Which Mode?

<CardGroup cols={2}>
  <Card title="Single Binary" icon="cube">
    **Choose this when:**
    - Crawling < 10K pages/day
    - You want minimal infrastructure
    - Running on a single server or serverless platform
    - Development and testing

    **Requires:** Only Meilisearch

    ```bash
    scrapix all --meilisearch-url http://meilisearch:7700
    ```
  </Card>
  <Card title="Distributed" icon="cubes">
    **Choose this when:**
    - Crawling > 10K pages/day
    - You need horizontal scaling
    - You want independent scaling of crawlers vs content workers
    - Running on Kubernetes or multi-server setup

    **Requires:** Meilisearch + Kafka/Redpanda

    ```bash
    scrapix api --brokers kafka:9092
    scrapix frontier --brokers kafka:9092
    scrapix crawler --brokers kafka:9092
    scrapix content --brokers kafka:9092
    ```
  </Card>
</CardGroup>

## Building the Docker Image

All deployment guides use the same multi-stage Dockerfile:

```bash
# Single binary (default target)
docker build -t scrapix .

# Specific service
docker build --target scrapix-api -t scrapix-api .
docker build --target scrapix-frontier-service -t scrapix-frontier .
docker build --target scrapix-worker-crawler -t scrapix-crawler .
docker build --target scrapix-worker-content -t scrapix-content .
```

Pre-built images are available at `ghcr.io/quentindequelen/scrapix:latest`.
