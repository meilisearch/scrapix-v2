# Environment Variables

All Scrapix services are configured through environment variables. When running in single-binary mode (`scrapix all`), all variables apply to the single process.

## API Server

| Variable | Default | Description |
|----------|---------|-------------|
| `HOST` | `0.0.0.0` | API server bind address |
| `PORT` | `8080` | API server port |
| `ENABLE_CORS` | `true` | Enable CORS headers |
| `API_KEY` | — | API authentication key (optional) |
| `JWT_SECRET` | `dev-jwt-secret-change-in-production` | JWT signing secret |
| `DATABASE_URL` | — | PostgreSQL URL for auth (optional) |
| `MAX_JOBS` | `10000` | Maximum tracked jobs |

## Message Queue

| Variable | Default | Description |
|----------|---------|-------------|
| `KAFKA_BROKERS` | `localhost:9092` | Comma-separated Kafka/Redpanda broker addresses |
| `KAFKA_GROUP_ID` | varies per service | Consumer group ID |

<Note>
In single-binary mode (`scrapix all`), set `--kafka-brokers` to use Kafka instead of in-process channels. If omitted, Scrapix uses zero-infrastructure in-memory channels.
</Note>

## Meilisearch

| Variable | Default | Description |
|----------|---------|-------------|
| `MEILISEARCH_URL` | `http://localhost:7700` | Meilisearch server URL |
| `MEILISEARCH_API_KEY` | `masterKey` | Meilisearch admin API key |
| `MEILISEARCH_INDEX` | `documents` | Default index UID |

## Cache (Redis / DragonflyDB)

| Variable | Default | Description |
|----------|---------|-------------|
| `REDIS_URL` | `redis://localhost:6380` | Redis/DragonflyDB connection URL |

## Analytics (ClickHouse)

| Variable | Default | Description |
|----------|---------|-------------|
| `CLICKHOUSE_URL` | `http://localhost:8123` | ClickHouse HTTP endpoint |
| `CLICKHOUSE_DATABASE` | `scrapix` | Database name |
| `CLICKHOUSE_USER` | `scrapix` | Username |
| `CLICKHOUSE_PASSWORD` | `scrapix` | Password |

<Note>
Setting `CLICKHOUSE_URL` automatically enables the analytics API at `/analytics/v0/pipes/`.
</Note>

## Frontier Service

| Variable | Default | Description |
|----------|---------|-------------|
| `DISPATCH_INTERVAL_MS` | `20` | Milliseconds between dispatch batches |
| `DISPATCH_BATCH_SIZE` | `2000` | URLs per dispatch batch |
| `DOMAIN_DELAY_MS` | `50` | Minimum delay between requests to same domain |
| `BLOOM_CAPACITY` | `10000000` | Bloom filter expected capacity per job |
| `BLOOM_FP_RATE` | `0.01` | Bloom filter false positive rate |
| `MAX_PENDING_PER_JOB` | `1000000` | Maximum pending URLs per job |
| `CONCURRENT_PER_DOMAIN` | `50` | Maximum concurrent requests per domain |
| `INSTANCE_ID` | — | Instance identifier for multi-instance frontier |

### Link Graph (optional)

| Variable | Default | Description |
|----------|---------|-------------|
| `ENABLE_LINKGRAPH` | `false` | Enable PageRank-based priority boosting |
| `LINKGRAPH_DAMPING` | `0.85` | PageRank damping factor |
| `LINKGRAPH_MAX_BOOST` | `50` | Maximum priority boost from PageRank |
| `LINKGRAPH_MAX_PAGES` | `10000000` | Maximum pages in link graph |
| `LINKGRAPH_COMPUTE_INTERVAL` | `300` | Seconds between PageRank recomputation |

### Recrawl (optional)

| Variable | Default | Description |
|----------|---------|-------------|
| `ENABLE_RECRAWL` | `false` | Enable automatic recrawling |
| `RECRAWL_MIN_AGE` | `3600` | Minimum seconds before recrawl |
| `RECRAWL_MAX_AGE` | `604800` | Maximum seconds before forced recrawl |
| `RECRAWL_MAX_URLS` | `10000000` | Maximum URLs in recrawl queue |

|----------|---------|-------------|

| `MAX_RETRIES` | `3` | Maximum retry attempts per URL |
| `MAX_DEPTH` | `100` | Maximum crawl depth |
| `MAX_BODY_SIZE_MB` | `10` | Maximum response body size in MB |
| `RESPECT_ROBOTS` | `true` | Respect robots.txt directives |
| `WORKER_ID` | — | Worker identifier |
| `ROCKSDB_PATH` | `./data/crawler-rocksdb` | RocksDB storage path |

### DNS Cache

### Link Graph Publishing

| Variable | Default | Description |
|----------|---------|-------------|

### Browser Rendering

| Variable | Default | Description |
|----------|---------|-------------|
| `BROWSER_RENDER` | `false` | Enable headless browser rendering |
| `BROWSER_RENDER_PATTERNS` | — | URL patterns to render (comma-separated globs) |
| `CHROME_PATH` | — | Path to Chrome/Chromium binary |
| `BROWSER_TIMEOUT` | `30` | Browser page load timeout in seconds |
| `BROWSER_CONCURRENCY` | `5` | Concurrent browser instances |
| `BROWSER_HEADLESS` | `true` | Run browser in headless mode |

## Content Workers

| Variable | Default | Description |
|----------|---------|-------------|
| `BATCH_SIZE` | `2000` | Meilisearch indexing batch size |
| `BATCH_TIMEOUT_SECS` | `5` | Maximum seconds to wait before flushing batch |
| `EXTRACT_CONTENT` | `true` | Enable readability content extraction |
| `CONVERT_MARKDOWN` | `true` | Convert extracted content to markdown |
| `DETECT_LANGUAGE` | `true` | Detect page language |
| `EXTRACT_SCHEMA` | `true` | Extract JSON-LD and microdata |
| `MIN_CONTENT_LENGTH` | `100` | Skip pages with less content (bytes) |
| `PUBLISH_TO_KAFKA` | `false` | Publish processed documents to Kafka |
| `PUBLISH_HISTORY` | `false` | Publish crawl history for incremental crawling |
| `SKIP_MEILISEARCH` | `false` | Skip Meilisearch indexing |

| Variable | Default | Description |
|----------|---------|-------------|

| `DEDUP_MINHASH_THRESHOLD` | `0.85` | MinHash Jaccard similarity threshold |
| `DEDUP_MAX_FINGERPRINTS` | `10000000` | Maximum stored fingerprints |
