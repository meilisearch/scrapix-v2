# Engines

Engine endpoints let you register multiple Meilisearch instances and set a default. This is useful when you have separate Meilisearch servers for different environments (staging, production) or different use cases.

<Note>
  Requires `DATABASE_URL` to be set. These endpoints are unavailable without PostgreSQL.
</Note>

## POST /engines

Register a new Meilisearch engine.

### Request Body

```json
{
  "name": "production",
  "url": "https://ms-prod.example.com",
  "api_key": "your-production-api-key",
  "description": "Production Meilisearch instance"
}
```

### Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Unique name for this engine |
| `url` | string | Yes | Meilisearch server URL |
| `api_key` | string | Yes | Meilisearch admin API key |
| `description` | string | No | Human-readable description |

### Response (201 Created)

```json
{
  "id": "a1b2c3d4-...",
  "account_id": "x1y2z3-...",
  "name": "production",
  "url": "https://ms-prod.example.com",
  "description": "Production Meilisearch instance",
  "is_default": false,
  "created_at": "2024-01-15T10:30:00Z",
  "updated_at": "2024-01-15T10:30:00Z"
}
```

## GET /engines/{id}

Get details for a specific engine.

### Response

Same as a single item from the list response.

---

## PATCH /engines/{id}

Update an engine's configuration.

### Request Body

```json
{
  "name": "production-v2",
  "description": "Updated production instance"
}
```

### Updatable Fields

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Engine name |
| `url` | string | Meilisearch URL |
| `api_key` | string | Meilisearch API key |
| `description` | string | Description |

### Response

Returns the full updated engine record.

---

## DELETE /engines/{id}

Delete a registered engine. Returns HTTP 204 (No Content) on success.

```bash
curl -X DELETE http://localhost:8080/engines/a1b2c3d4-...
```

<Warning>
  You cannot delete the default engine. Set another engine as default first.
</Warning>

---

## POST /engines/{id}/default

Set an engine as the default. The default engine is used when crawl configs don't specify a `meilisearch` block.

### Response

```json
{
  "id": "a1b2c3d4-...",
  "name": "production",
  "is_default": true,
  "message": "Engine set as default"
}
```

---

## GET /engines/{id}/indexes

List all indexes in a registered engine. Proxies the request to the engine's Meilisearch instance.

### Response

```json
{
  "results": [
    {
      "uid": "my-docs",
      "primaryKey": "uid",
      "numberOfDocuments": 1250,
      "createdAt": "2024-01-10T00:00:00Z",
      "updatedAt": "2024-01-15T10:30:00Z"
    }
  ],
  "total": 1,
  "limit": 20,
  "offset": 0
}
```

## Use Cases

- **Multi-environment** — Register staging and production Meilisearch instances, switch between them
- **Team isolation** — Different teams use different engines with separate API keys
- **Console integration** — The Scrapix Console uses engines to let users pick their target Meilisearch when creating crawls
