# Authentication

When `DATABASE_URL` is configured, Scrapix enables a full authentication system with user accounts, API keys, session-based auth, and billing tier management.

<Note>
  All auth endpoints return 404 when `DATABASE_URL` is not set. Without PostgreSQL, the API runs in open mode with no authentication.
</Note>

## POST /auth/signup

Create a new user account. Automatically creates a default account and assigns the user as owner.

### Request Body

```json
{
  "email": "user@example.com",
  "password": "secure-password-123",
  "full_name": "Jane Doe"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `email` | string | Yes | Unique email address |
| `password` | string | Yes | Password (hashed with Argon2) |
| `full_name` | string | No | Display name |

### Response (201 Created)

Sets a `scrapix_session` HttpOnly cookie and returns the user profile:

```json
{
  "id": "a1b2c3d4-...",
  "email": "user@example.com",
  "full_name": "Jane Doe",
  "account": {
    "id": "x1y2z3-...",
    "name": "Jane Doe's account",
    "tier": "free",
    "active": true,
    "role": "owner"
  }
}
```

---

## POST /auth/login

Authenticate and receive a session cookie.

### Request Body

```json
{
  "email": "user@example.com",
  "password": "secure-password-123"
}
```

### Response

Sets a `scrapix_session` HttpOnly cookie (7-day expiration) and returns the user profile (same format as signup).

---

## POST /auth/logout

Clear the session cookie.

### Response

```json
{
  "message": "Logged out"
}
```

## GET /auth/me

Get the authenticated user's profile.

**Requires:** Session cookie

### Response

```json
{
  "id": "a1b2c3d4-...",
  "email": "user@example.com",
  "full_name": "Jane Doe",
  "account": {
    "id": "x1y2z3-...",
    "name": "My Company",
    "tier": "pro",
    "active": true,
    "role": "owner"
  }
}
```

## API Keys

API keys provide programmatic access to protected endpoints. They are sent via the `x-api-key` header.

### GET /account/api-keys

List all API keys for the authenticated account.

**Requires:** Session cookie

### Response

```json
[
  {
    "id": "k1l2m3-...",
    "name": "Production Key",
    "prefix": "sk_live_abc1",
    "active": true,
    "last_used_at": "2024-01-15T10:30:00Z",
    "created_at": "2024-01-01T00:00:00Z"
  }
]
```

<Note>
  The full key is never shown after creation. Only the 12-character prefix is stored for display.
</Note>

### POST /account/api-keys

Create a new API key.

**Requires:** Session cookie

### Request Body

```json
{
  "name": "Production Key"
}
```

### Response (201 Created)

```json
{
  "id": "k1l2m3-...",
  "name": "Production Key",
  "prefix": "sk_live_abc1",
  "key": "sk_live_abc123def456ghi789jkl012mno345pqr678stu901vwx234yz"
}
```

<Warning>
  The `key` field is only returned once at creation time. Store it securely — it cannot be retrieved again.
</Warning>

### PATCH /account/api-keys/{id}

Revoke (deactivate) an API key.

**Requires:** Session cookie

### Request Body

```json
{
  "active": false
}
```

## OAuth 2.1

Scrapix also supports OAuth 2.1 with PKCE for third-party integrations and MCP clients. OAuth provides:

- **Dynamic client registration** — Clients register themselves automatically
- **PKCE (S256)** — Secure authorization code flow without client secrets
- **Refresh token rotation** — Automatic token renewal with revocation on reuse
- **Bearer tokens** — For authenticating MCP and API requests

| Endpoint | Description |
|----------|-------------|
| `GET /.well-known/oauth-authorization-server` | OAuth metadata discovery |
| `POST /oauth/register` | Register an OAuth client |
| `GET/POST /oauth/authorize` | Authorization code flow |
| `POST /oauth/token` | Exchange code or refresh token |
| `POST /oauth/revoke` | Revoke a token |

See the [OAuth 2.1 Guide](/guides/oauth) for the complete flow and integration details.

## Session Cookie Details

| Property | Value |
|----------|-------|
| Name | `scrapix_session` |
| Type | JWT (HS256) |
| HttpOnly | Yes |
| SameSite | Lax |
| Expiration | 7 days |
| Path | `/` |
| Contains | User ID, email |

The JWT is signed with `JWT_SECRET` (set via environment variable or `--jwt-secret` flag).
