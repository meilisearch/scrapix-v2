# API Reference

The Scrapix API is a REST API built with [Axum](https://github.com/tokio-rs/axum) that manages distributed web crawling jobs, single-page scraping, and real-time event streaming.

## Base URL

```
http://localhost:8080
```

In production, replace with your deployment URL.

## Authentication

Authentication is **optional** — when `DATABASE_URL` is not configured, all endpoints are open. When enabled, protected endpoints require one of:

| Method | Header / Cookie | Example |
|--------|----------------|---------|
| API Key | `x-api-key` header | `x-api-key: sk_live_abc123...` |
| Session | `scrapix_session` cookie | Set automatically after login |

See [Authentication Guide](/guides/authentication) for setup details.

## Request Format

All request bodies use JSON (`Content-Type: application/json`).

## Response Format

All responses return JSON. Successful responses return the data directly. Errors follow a consistent format:

```json
{
  "error": "Human-readable error message",
  "code": "machine_readable_code",
  "details": null
}
```

### Error Codes

| Code | HTTP Status | Description |
|------|-------------|-------------|
| `not_found` | 404 | Resource does not exist |
| `bad_request` | 400 | Malformed request |
| `validation_error` | 400 | Invalid field values |
| `unauthorized` | 401 | Missing or invalid credentials |
| `conflict` | 409 | Resource already exists |
| `tier_limit_exceeded` | 429 | Billing tier limit reached |
| `queue_error` | 500 | Failed to publish to message queue |
| `timeout` | 500 | Operation timed out |

## Rate Limiting

When authentication is enabled, requests are rate-limited per account based on billing tier:

| Tier | Requests/Minute |
|------|----------------|
| Free | 10 |
| Starter | 60 |
| Pro | 300 |
| Enterprise | 1,000 |

See [Rate Limiting](/api-reference/rate-limiting) for details on headers, 429 responses, and handling.

## OpenAPI & Interactive Docs

The API serves an OpenAPI 3.1 specification and interactive documentation:

- **`GET /openapi.json`** — Machine-readable OpenAPI spec (used by MCP server)
- **`GET /docs`** — Interactive API explorer powered by Scalar
